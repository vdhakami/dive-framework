## New DIVE Framework Addresses 70% AI Project Failure Rate

*Open-source project management methodology introduces parallel exploration, quantitative kill criteria, and compute tracking for AI initiatives*

**LONDON, UK** — Vahid Hakami today announced the release of the DIVE Framework, a new project management methodology purpose-built for artificial intelligence projects. The framework addresses a critical gap exposed by PMI research showing that over 70% of AI initiatives fail to deliver expected value — a failure rate the author attributes to using deterministic project management methods for probabilistic AI work.

"PMBOK, PRINCE2, Agile, and SAFe were all designed for deterministic software engineering, where requirements can be specified upfront and quality is binary," said Hakami. "AI doesn't work that way. Models degrade over time. The winning approach is unknown at project start. The dominant cost isn't engineering hours — it's GPU compute and API tokens. DIVE is the first framework built from the ground up for these realities."

DIVE stands for Diagnose → Iterate → Validate → Evolve, a four-phase lifecycle supported by seven core principles and 14 modular templates.

### Key Innovations

DIVE introduces several mechanisms absent from existing frameworks such as PMI's CPMAI, Microsoft's TDSP, and MLOps:

- **Parallel Exploration Tracks:** Teams run 3-5 independent experiments simultaneously, each with a fixed compute budget, rather than pursuing approaches sequentially.
- **Metric Contracts:** Signed quantitative thresholds replace vague requirements documents, reducing scope creep and stakeholder misalignment.
- **Quantitative Kill Criteria:** Pre-defined metrics determine when to kill or redesign a track, treating early termination as a resource preservation success rather than a failure.
- **Compute as a Tracked Resource:** GPU time and API costs are monitored alongside schedule and budget as first-class project constraints.
- **Continuous Governance Protocol:** A living Governance Card tracks data drift, concept drift, and performance degradation for every production model.
