## 1:45–3:30 — The 7 Principles (brief)

**Visual:** Each principle appears as a card on screen

**Audio:**
**1. Data First, Code Second.** No AI project begins with architecture. It begins with a data audit. If the data doesn't contain signal, no model can save you.

**2. Parallel Exploration Beats Sequential Certainty.** The winning approach is unknown at project start. DIVE mandates 3-5 parallel experiment tracks, each with a fixed compute budget. Tracks converge when metrics are validated, not when a calendar says so.

**3. Metric Contracts Replace Requirements Documents.** Instead of vague requirements, DIVE uses signed agreements with precise thresholds. 'F1 ≥ 0.87 on held-out test set, latency ≤ 200ms.' That's a requirement. Everything else is a wish.

**4. Fail Fast, Fail Quantitatively.** Quantitative kill criteria are defined upfront. Hit a gate without meeting metrics? The project is killed or redesigned. A kill is a success — it preserved resources.

**5. Governance Is Continuous, Not Periodic.** AI models degrade continuously. DIVE mandates permanent monitoring and a living Governance Card for every production model.

**6. Human-in-the-Loop Is a Design Constraint.** Three escalation tiers designed before deployment: Advisory, Assisted, and Automated. You design the safety net before you need it.

**7. Compute Is a First-Class Resource.** GPU time and API costs dominate AI budgets. DIVE tracks compute burn rate alongside schedule and budget. Each track has a fixed compute budget. Exceed it without results? Redesign."

## 3:30–4:15 — How It Works in Practice

**Visual:** Timeline diagram showing a sample 12-week project using DIVE

**Audio:**
"Let's see how this plays out. A typical DIVE project runs 8-16 weeks.

"In Diagnose, you validate the problem, audit your data, establish a baseline, and sign a Metric Contract with all stakeholders.

"In Iterate, you launch 3-5 parallel experiments. Each week, you compare results on a shared dashboard. Tracks that miss their compute budget without progress are killed. The winning approach emerges from data, not from politics.

"In Validate, you do a rigorous hold-out evaluation, bias audit, and stakeholder demo. If metrics are met, you get a Go decision.

"In Evolve, the model is deployed with continuous monitoring. Drift is tracked. Alerts fire automatically. Quarterly governance reviews ensure continued alignment."
