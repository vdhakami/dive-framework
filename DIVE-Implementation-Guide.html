P1 Diagnose — Validate Feasibility

1

Problem Framing Workshop

Gather all stakeholders. Ask: "If this AI works perfectly, what changes in the business?" Write the answer as your North Star. Determine problem type (classification, generation, regression, etc.).

⏰ 2-3 days
2

Data Landscape Audit

List every data source. For each: type, volume, freshness, owner, access method. Score quality 1-5 on: completeness, consistency, accuracy, timeliness, relevance, accessibility.

⏰ 3-5 days
3

Feasibility Baseline

Train the simplest possible model (logistic regression, zero-shot LLM) on available data. If baseline ≤ random heuristic, flag for redesign. Document baseline metrics.

⏰ 2-4 days
4

Metric Contract Negotiation

Define primary metrics (target + minimum), secondary metrics (warning thresholds), constraint gates (latency, cost, model size), test set spec, and fairness requirements. All parties sign.

⏰ 2-3 days
5

Ethics Screening

Run the 10-question AI Ethics Checklist. Classify project as Low / Medium / High risk. Document mitigations. High risk requires full board review before proceeding.

⏰ 1 day
6

Risk Register + Build/Buy

Create AI Risk Register (7 categories: data, model, operational, ethical, governance, drift, security). Assess build vs. buy. Document in Project Charter.

⏰ 1-2 days
🚨 EXIT GATE: Feasibility Review
  • Data exists, accessible, and signal > noise floor
  • Metric Contract signed by ALL stakeholders
  • Baseline model > random heuristic
  • Ethics screening passed (or mitigations documented)

P2 Iterate — Parallel Experimentation

Parallel Track Architecture

1

Design Experiment Plan

Create 3-5 parallel tracks. Each track explores a different approach (different algorithm, prompt strategy, or data representation). Each gets a fixed compute budget.

⏰ 1-2 days
2

Allocate Resources

Assign team members per track. Provision GPU/API budgets. Set up experiment tracking (MLflow, W&B, or simple spreadsheet).

⏰ 1 day
3

Execute + Track

All tracks run simultaneously. Each logs to Experiment Log (T4). Weekly all-track sync meeting compares results on Comparison Dashboard (T5).

⏰ 3-6 weeks
4

Mid-Phase Pivot

At midpoint: kill clearly underperforming tracks. Reallocate budget to promising ones. Add new tracks if justified by interim findings.

⏰ Week 3-4
5

Select Candidate

Compare final metrics vs. Metric Contract. Select top track(s). Criteria: meets thresholds, reproducible, documented, production-feasible.

⏰ 2-3 days
🚨 EXIT GATE: Candidate Selection
  • At least one track meets ALL Metric Contract thresholds
  • Top candidate reproducible + documented
  • Compute budget variance < 20%
  • Risk register updated with model-specific risks
T4: Experiment Log T5: Track Dashboard T14: Compute Tracker

P3 Validate — Prove It Works

5 Validation Dimensions

1

Hold-Out Evaluation

Evaluate on held-out test set (never seen during Iterate). Report every metric from the Metric Contract. Compare achieved vs. target vs. minimum.

⏰ 2-3 days
2

Robustness Testing

Test on: missing data, out-of-distribution inputs, adversarial examples, edge cases. Document failure modes and error rates.

⏰ 2-3 days
3

Bias Audit

Run disparate impact analysis across protected attributes (race, gender, age). Measure demographic parity, equal opportunity, equalized odds. Remediate if thresholds exceeded.

⏰ 1-2 days
4

Performance Benchmark

Measure latency (p50/p95/p99), throughput, memory, model size, cost per inference. Compare against constraint gates in Metric Contract.

⏰ 1-2 days
5

Stakeholder Demo

Present results. Show model behavior on representative examples. Walk through edge cases. Obtain signed Validation Certificate.

⏰ 1 day
🚨 EXIT GATE: Validation Sign-Off
  • ALL Metric Contract thresholds met in production-like environment
  • Bias audit clean or mitigations accepted
  • Edge case handling documented
  • Stakeholder demo completed with signed certificate
T6: Evaluation Report T10: Deployment Manifest

P4 Evolve — Continuous Governance

Drift Response Decision Tree

1

Production Deployment

Execute per Deployment Manifest. Strategy: Blue/Green, Canary, or Rolling. Configure autoscaling, timeout, retry, fallback. Document rollback.

⏰ 2-5 days
2

Monitoring Setup

Configure automated monitoring for data drift, concept drift, performance drift. Set alert thresholds (Yellow/Orange/Red). Define escalation paths.

⏰ 1-2 days
3

Governance Card

Create the Model Governance Card (T8). Document: identity, intended use, performance baseline, fairness metrics, monitoring config, dependencies.

⏰ 1 day
4

Continuous Monitoring

Monitor dashboards daily. Respond to alerts per Drift Response Protocol. Log all incidents in Drift Alert Log (T11).

⏰ Ongoing
5

Quarterly Review

Every 3 months: full governance review. Update Governance Card. Assess if model still meets business needs. Decision: continue, retrain, replace, or decommission.

⏰ Quarterly
T8: Governance Card T11: Drift Alert Log

📝 How to classify your project:

Score yourself 1-4 on each factor, average the score, round to nearest integer:

  • Autonomy: 1=Advisory, 2=Assisted, 3=Automated, 4=Full autonomy
  • Impact: 1=Inconvenience, 2=Financial loss, 3=Health/safety, 4=Life-threatening
  • Regulation: 1=None, 2=Some, 3=Heavily regulated, 4=Multiple jurisdictions
  • Scales: 1=<100 users, 2=100-10K, 3=10K-1M, 4=>1M

Quick-Start Master Checklist

Check off each item as you complete it. Click the checkbox to mark done.

■ Diagnose

  • Complete Problem Framing Workshop with all stakeholders
  • Audit all data sources (quality scores > 3.5 on 6 dimensions)
  • Train feasibility baseline model (must beat random heuristic)
  • Negotiate and sign Metric Contract with ALL stakeholders
  • Complete AI Ethics Screening Checklist
  • Create initial AI Risk Register
  • Complete Build vs. Buy Assessment
  • Pass Feasibility Gate (Go/No-Go decision)

■ Iterate

  • Design 3-5 parallel experiment tracks
  • Allocate compute budget per track (Track in T14)
  • Set up experiment tracking tool (MLflow / W&B / spreadsheet)
  • Execute all tracks simultaneously
  • Weekly all-track sync meetings (use T5 dashboard)
  • Mid-phase pivot: kill underperformers, reallocate budget
  • Select top candidate (meets Metric Contract thresholds)
  • Pass Candidate Selection Gate

■ Validate

  • Run hold-out evaluation on unseen test set
  • Complete robustness and edge case testing
  • Run bias audit (disparate impact analysis)
  • Complete performance benchmark (latency, throughput, cost)
  • Present stakeholder demo and obtain signed Validation Certificate
  • Pass Validation Sign-Off Gate

■ Evolve

  • Deploy to production (with rollback plan)
  • Configure monitoring dashboard (data/concept/performance drift)
  • Create Model Governance Card (T8)
  • Establish drift baselines (record initial distributions)
  • Set up feedback pipeline for production data
  • Schedule quarterly governance reviews
